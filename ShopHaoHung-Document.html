ShopHaoHung là một ứng dụng thương mại điện tử hiện đại, được xây dựng theo mô hình SPA (Single Page Application) với kiến trúc App Router của Next.js. Dự án phục vụ hai nhóm người dùng chính: khách hàng mua sắm và quản trị viên quản lý cửa hàng.

Toàn bộ dữ liệu sản phẩm hiện được lưu qua localStorage với seed từ file badminton-products.json, cho phép trải nghiệm CRUD đầy đủ ngay trên trình duyệt mà không cần backend thực.

Tính Năng Chính

🛒

Khách Hàng (User)

  • Duyệt sản phẩm với tìm kiếm, lọc, sắp xếp
  • Lọc theo danh mục, giá, trạng thái tồn kho
  • Trang chi tiết sản phẩm, hình ảnh lớn
  • Giỏ hàng lưu riêng theo tài khoản đã đăng nhập
  • Quy trình checkout & xác nhận đơn hàng
  • Đăng nhập / Đăng ký / Hồ sơ cá nhân
👨‍💼

Quản Trị Viên (Admin)

  • Dashboard tổng quan: doanh thu, đơn hàng, KPIs
  • CRUD sản phẩm – tạo, sửa, xóa realtime
  • Quản lý đơn hàng với bộ lọc trạng thái
  • Quản lý người dùng (Admin / User)
  • Báo cáo & phân tích biểu đồ
  • Trang tải file & quản lý tài nguyên
🔐

Bảo Mật & Xác Thực

  • JWT token + Cookie httpOnly
  • Session tự phục hồi khi reload trang
  • Role-based access control (RBAC)
  • Middleware bảo vệ route, AdminGuard component
  • Hydration flag chống flickering
  • Header cập nhật tức thì sau đăng nhập/logout
⚡

Performance & SEO

  • Image optimization: next/image, WebP/AVIF
  • ISR trên shop listing (revalidate: 3600s)
  • SSG cho trang tĩnh (About, Terms)
  • Dynamic imports giảm bundle size
  • Web Vitals tracking: LCP, FID, CLS
  • SEO metadata, Open Graph, sitemap tự động

Kiến Trúc Hệ Thống

Browser → Next.js 15 App Router │ ├── Pages (App Router Groups) │ ├── (auth)/ → /auth/login, /auth/signup │ ├── (shop)/ → /shop, /shop/[id] │ └── (dashboard)/ → /dashboard/** [Admin only] │ ├── React Components │ ├── common/ → Button, Input, Skeleton │ ├── layout/ → Header, Footer, RootLayout │ └── product/ → ProductCard │ ├── State Management │ ├── cartStore (Zustand) → giỏ hàng theo user │ └── uiStore (Zustand) → toasts, modals │ └── Data Layer └── lib/product-storage.ts → localStorage key: shophaohung.products → Seed: lib/badminton-products.json

Phân Lớp Dữ Liệu

Dự án chia rõ ba lớp lưu trữ trạng thái:

Global State – Zustand

  • cartStore: items, total, addItem, removeItem, updateQuantity, clearCart
  • uiStore: toasts, addToast, removeToast, clearToasts

Local State – React useState

  • Form inputs, UI toggles
  • Filter tạm thời trên trang shop
  • Product form state ở dashboard

Client Storage – localStorage

  • shophaohung.products: danh mục sản phẩm đồng bộ admin ↔ shop
  • Session người dùng (token, thông tin user)
  • Giỏ hàng riêng theo user.id

Luồng CRUD Sản Phẩm

  • Admin tạo/sửa/xóa → ghi vào localStorage
  • Seed tự động từ JSON khi lần đầu truy cập
  • Storage event đồng bộ dashboard & shop ngay lập tức

API & Dữ Liệu

Lưu ý: Hiện tại sản phẩm được lưu qua localStorage (key: shophaohung.products). API routes vẫn tồn tại để tương thích, nhưng nguồn dữ liệu thực tế là client-side storage.

Nhóm Endpoints Chính

Method Endpoint Mô tả Auth?
GET /api/products Danh sách sản phẩm (search, filter, sort, page) Không
GET /api/products/:id Chi tiết một sản phẩm Không
POST /api/products Tạo sản phẩm mới Admin
PUT /api/products/:id Cập nhật sản phẩm Admin
DEL /api/products/:id Xóa sản phẩm Admin
POST /api/auth/login Đăng nhập – trả JWT + set cookie Không
POST /api/auth/signup Đăng ký tài khoản mới Không
POST /api/auth/logout Đăng xuất – clear cookie & session Có
POST /api/orders Tạo đơn hàng mới User
POST /api/upload Upload ảnh (avatar / sản phẩm) Admin
POST /api/vitals Ghi nhận Web Vitals metrics Không

Triển Khai Vercel qua GitHub Actions

Workflow nằm ở .github/workflows/deploy-to-vercel.yml. Cần thiết lập 3 GitHub Secrets:

GitHub Secrets cần thiết lập

  • VERCEL_TOKEN – từ Vercel Account Settings → Tokens
  • VERCEL_ORG_ID – trong file .vercel/project.json
  • VERCEL_PROJECT_ID – trong file .vercel/project.json

Quy trình deploy

  • Push code → Preview deploy tự động chạy
  • Kiểm tra URL preview trong tab Actions
  • Vào Actions → Run workflow → chọn production=true để deploy production
  • URL production: shop-haohung.vercel.app

CI/CD Pipeline (GitHub Actions)

Push to GitHub │ ├── Lint (ESLint) ├── Type check (tsc --noEmit) ├── Build verification (npm run build) ├── Test suite (Jest + Playwright) └── Deploy ├── Preview → mọi push / PR └── Production → manual trigger (production=true)
